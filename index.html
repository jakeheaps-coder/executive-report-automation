Weekly Executive Report Automation

01 — Pipeline

4 Steps, 100% Inside Domo

Every Friday at 9am, a Domo Workflow fires a TIMER trigger that executes four Code Engine functions in sequence -- collect, generate, grade, deliver.

02 — Data Sources

Three Sources of Truth

The pipeline reads live card data from the Domo dashboard via API, plus two supplementary Domo datasets uploaded from SharePoint.

Domo Dashboard

41 cards read directly via the Domo API -- KPI values, gauges, notebook text, and table data. No screenshots, no vision model. Just the raw data a human sees.

Primary #1
  • Notebook cards split into THIS WEEK / PRIOR WEEKS
  • KPI card definitions + summary values
  • Table card datasets queried via SQL
  • Card metadata for all 41 cards

MLT Report

Marketing Leadership Team report uploaded as a Domo dataset. Contains initiative status updates across all functional areas.

Primary #2
  • Product & Integration updates
  • Paid Search, SEO/GEO status
  • AI & Outbound initiatives
  • Events, Comms & Brand, Regional

Marketing Calendar

Excel spreadsheet uploaded as a Domo dataset. Contains event dates, registration counts, and activity timelines.

Secondary
  • Event dates & registrations
  • Activity timeline data
  • Campaign milestones
  • Calendar section input only
03 — Ralph's Loop

Grade Until Perfection

Every report is graded against a 5-criteria rubric by DomoGPT. If it doesn't pass, the AI refines it with specific feedback and tries again. Up to 10 rounds. We don't lower the bar -- we raise the report.

The Loop
Named after Ralph Waldo Emerson: "That which we persist in doing becomes easier -- not that the task itself has become easier, but that our ability to perform it has improved."
Step A

Grade Draft

DomoGPT scores the report against 5 criteria with specific line-level feedback

Step B

Pass or Fail?

Score >= 80 on every criterion means pass. Any criterion below = fail the whole report

Step C

Refine with Feedback

Failed criteria become specific instructions: "fix X in paragraph Y, the correct number is Z"

Step D

Loop Back

New draft replaces old. Re-grade from scratch. Repeat until perfect or 10 attempts

Up to 10 rounds -- we never lower the bar

DomoGPT Generation

The pipeline uses DomoGPT (Claude/Anthropic) to synthesize all source data into a structured executive report using an 11-section system prompt.

  • Model: DomoGPT Medium v1.1 (Anthropic)
  • Temperature: 0.3 (consistent, accurate)
  • Max tokens: 8,000
  • Full 11-section system prompt

DomoGPT Grading

A separate DomoGPT call grades the report against 5 criteria with granular scoring and specific fix instructions for each failure.

  • 5 criteria scored individually
  • Each criterion must hit >= 80
  • Specific feedback per failed criterion
  • Draft stored in AppDB between rounds
04 — Deliverables

Email with PDF, Ready for CMO

The pipeline delivers a branded HTML email with the full report inline and a PDF attachment -- sent directly to the review team via Domo's email API.

HTML Email

Branded HTML email with the full report rendered inline. Executive summary, key wins, risks, calendar -- all formatted for quick consumption in any email client.

PDF Attachment

The full report rendered as a PDF using pdf-lib in Code Engine. Attached directly to the email as a real file -- not a link, not a redirect.

AppDB Archive

Every report, grade, and source snapshot is archived in Domo AppDB. Provides 4-week trend context for the next generation cycle.

06 — Infrastructure

Domo-Native, Zero External Dependencies

The entire pipeline runs inside Domo -- Workflows for orchestration, Code Engine for compute, AppDB for state, and DomoGPT for AI. No GCP, no external APIs.

Domo Workflows

Pipeline orchestration

  • 4 Code Engine steps in sequence
  • Ralph's Loop on grade_and_refine
  • TIMER trigger: Fridays 9am MST
  • Manual trigger for ad-hoc runs

Code Engine

Serverless JavaScript functions

  • 4 packages, 1 function each
  • collect_data (41 cards + datasets)
  • generate_report (DomoGPT synthesis)
  • grade_and_refine (Ralph's Loop)
  • format_and_deliver (PDF + email)

State Management

AppDB collections

  • State collection: drafts, grades, final
  • Archive collection: weekly history
  • Data passes between steps via AppDB
  • 4-week trend context for generation
  • FileSet for PDF storage
